# Contributing to neubrutalism.com

Thanks for wanting to improve the definitive guide to neubrutalism! This is a community resource and contributions are welcome.

## How to contribute

1. **Fork** this repository
2. **Create a branch** for your change (`git checkout -b my-improvement`)
3. **Make your changes** -- see guidelines below
4. **Submit a pull request** with a clear description of what you changed and why

## What we're looking for

- **Content improvements**: Corrections, better explanations, additional historical context, new examples
- **Design refinements**: Better responsiveness, accessibility improvements, visual polish
- **New sections**: If you think something important is missing, propose it
- **Translations**: We'd love to have this resource in more languages
- **Bug fixes**: Broken layouts, typos, dead links

## Guidelines

- **Vanilla only.** No frameworks, no build steps, no JavaScript dependencies. HTML + CSS + minimal vanilla JS.
- **Mobile first.** Everything must work down to 250px viewport width.
- **Accessibility matters.** Maintain WCAG AA compliance. Test focus states, contrast ratios, and screen reader behavior.
- **Practice what we preach.** This site IS neubrutalism. Any design changes should exemplify the style.
- **Keep it authoritative.** This is a reference, not a blog post. Cite sources where possible.

## Code style

- Use CSS custom properties for all design tokens
- Semantic HTML5 elements
- Meaningful class names
- Comment sections clearly

## Questions?

Open an issue! We're happy to discuss ideas before you invest time in a PR.
